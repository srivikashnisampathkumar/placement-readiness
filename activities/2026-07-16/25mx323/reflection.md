# Reflection

## What I Learned

This activity introduced me to the standard Git and GitHub workflow used in collaborative software development.

I learned how to:

- Fork a repository
- Clone a repository to my local machine
- Create files in the correct directory
- Stage and commit changes
- Push commits to my fork
- Create a Pull Request

## Challenges

Initially, understanding the difference between a fork and a clone was slightly confusing. After completing the workflow, I understood that a fork creates my own copy of a repository, while cloning downloads that repository to my local system.

## Key Takeaways

- Always work only inside the assigned folder.
- Use meaningful commit messages.
- Verify changes before pushing.
- Follow repository contribution guidelines carefully.

## Conclusion

This exercise strengthened my understanding of version control and collaborative development practices, which are essential skills for software engineering and open-source contributions.
