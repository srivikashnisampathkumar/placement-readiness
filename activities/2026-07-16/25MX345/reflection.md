Mission Tata 1 provided practical exposure to Git and GitHub, which are essential tools in modern software development. Instead of learning only the theory, I completed the entire workflow myself.
What I learned during this mission, I learned how to:

  Fork a repository from GitHub.
  Clone a repository to my computer.
  Organize files inside the correct project structure.
  Track changes using Git.
  Stage and commit changes with meaningful commit messages.
  Push commits to my GitHub repository.
  Create a Pull Request to contribute changes back to the original repository.
  Challenges Faced

Initially, I was slightly confused about the difference between Fork, Clone, and Pull Request. After practicing each step, I understood how these operations work together in a collaborative development environment.

Another important lesson was ensuring that I modified only my assigned folder to avoid affecting other contributors' work.

Key Takeaways:
Git helps track every change made to a project.
GitHub enables collaboration among multiple developers.
Pull Requests provide a structured review process before merging code.
Small and meaningful commits improve project history.
Version control is an essential skill for internships and software engineering careers.

Conclusion:
This mission improved my confidence in using Git and GitHub. I now understand the basic collaboration workflow and feel prepared to contribute to larger software projects in the future.
