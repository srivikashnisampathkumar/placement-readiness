# Git Workflow Practice - Placement Readiness

## Overview

This activity focuses on learning and practicing the basic Git and GitHub workflow rather than developing a software application.

The objective is to understand how to contribute to a shared repository using industry-standard version control practices.

---

## Objectives

- Fork the repository to a personal GitHub account.
- Clone the forked repository locally.
- Create a personal submission folder.
- Add a profile file.
- Commit the changes.
- Push the changes to the fork.
- Create a Pull Request to the main repository.

---

## Git Workflow Followed

1. Forked the original repository.
2. Cloned the fork locally.
3. Created the folder:

```

activities/2026-07-16/25mx323/

```

4. Added the required markdown files.
5. Staged the files using Git.
6. Created a commit with an appropriate message.
7. Pushed the changes to the fork.
8. Opened a Pull Request to the original repository.
