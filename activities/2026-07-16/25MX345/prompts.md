AI Prompts Used During Mission

The following prompts were used to understand Git concepts and complete the assignment.

Prompt 1

Prompt:

Explain the complete Git workflow from Fork to Pull Request with simple examples.

Purpose:
To understand how developers collaborate using GitHub.

Prompt 2

Prompt:

Give the Git commands required to clone a repository, stage files, commit changes, and push them to GitHub.

Purpose:
To perform the required Git operations correctly.

Prompt 3

Prompt:

Explain the difference between Git and GitHub with examples.

Purpose:
To understand the role of version control and remote repositories.

Prompt 4

Prompt:

How do I resolve common Git errors such as authentication issues, merge conflicts, and rejected pushes?

Purpose:
To learn how to troubleshoot common Git problems.

Prompt 5

Prompt:

Generate professional README.md, reflection.md, and prompts.md files for a Git & Collaboration assignment.

Purpose:
To prepare clear and organized documentation for submission.

Summary

AI was used as a learning assistant to understand Git concepts, clarify commands, and improve documentation quality. All Git operations, including forking, cloning, committing, pushing, and creating the Pull Request, were performed manually.
